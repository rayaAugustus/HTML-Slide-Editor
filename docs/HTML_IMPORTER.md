# HTML Importer

## Purpose

The HTML importer lets users bring HTML generated by other AI tools into HTML Slide Editor.

This is a major product feature because users may generate HTML from Claude, Codex, GPT, Gemini, v0, screenshots, or other tools, then need a stable visual editor to refine the result.

## Important Boundary

Do not promise perfect conversion from arbitrary HTML to fully editable slide elements.

Arbitrary HTML can contain:

- nested DOM
- external CSS
- external fonts
- JavaScript
- SVG
- Canvas
- complex Grid/Flex layouts
- responsive breakpoints
- animations
- absolute and relative positioning mixed together

A personal-developer project should handle this in layers.

## Import Modes

### Mode A: Sandbox Import

P0 mode.

```text
.html file -> read text -> create HtmlElement -> render inside slide
```

User can:

- import the HTML as a slide
- move/resize the whole imported content
- export it with the deck

User cannot:

- edit every internal DOM node visually
- guarantee perfect external asset loading

### Mode B: AI-assisted Conversion

P1/P2 mode.

```text
.html file -> AI or parser analyzes DOM/CSS -> SlideElement[]
```

This attempts to convert major blocks into editable elements:

- headings -> TextElement
- paragraphs -> TextElement
- cards/backgrounds -> ShapeElement
- images -> ImageElement

### Mode C: Hybrid Import

Long-term practical mode.

```text
complex HTML background preserved as HtmlElement
important text/shapes extracted as editable elements
```

This provides partial editability without requiring perfect conversion.

## P0 Implementation

Function:

```ts
function importHtmlAsSandbox(html: string): SlideDoc
```

Output:

```ts
const slide: SlideDoc = {
  id: crypto.randomUUID(),
  name: "Imported HTML",
  width: 960,
  height: 540,
  background: { type: "solid", color: "#ffffff" },
  elements: [
    {
      id: crypto.randomUUID(),
      type: "html",
      x: 0,
      y: 0,
      width: 960,
      height: 540,
      zIndex: 0,
      html: sanitizedHtml,
      editableLevel: "none",
      sandbox: true
    }
  ]
};
```

## File Input Requirements

Support:

- `.html`
- `.htm`

Optional later:

- pasted HTML
- zip package with assets
- URL import

## Security Requirements

Imported HTML is untrusted.

P0 should:

- strip `<script>` tags
- remove inline event handlers such as `onclick`
- avoid executing imported JavaScript
- render imported HTML in a sandboxed iframe if possible

Recommended sandbox:

```html
<iframe sandbox="allow-same-origin"></iframe>
```

Avoid `allow-scripts` for imported content.

## Sanitization Rules

At minimum:

- remove `<script>...</script>`
- remove `on*="..."` attributes
- optionally remove external scripts
- optionally warn when external assets are detected

## Export Behavior

When exporting a deck containing HtmlElement:

- include the sanitized HTML content
- keep it inside a bounded slide container
- do not re-enable scripts
- preserve width/height scaling as much as possible

## AI Conversion Later

A later function may be added:

```ts
async function convertHtmlToSlideDoc(html: string): Promise<SlideDoc>
```

But this should not block the P0 editor.

## Success Criteria for P0

The P0 importer is successful if:

1. user can upload an HTML file
2. a new slide appears
3. the imported HTML is visible
4. the imported slide can be exported
5. scripts are not executed

## Product Note

Sandbox import is not a weakness. It is the correct first layer.

Perfect HTML-to-editable-elements conversion is a hard problem and should be treated as an enhancement, not a foundation.
