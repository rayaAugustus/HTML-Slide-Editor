# Product Spec

## Product Name

HTML Slide Editor

## One-line Description

A lightweight HTML slide editor that supports PPT-like editing, HTML import, and stable template-based slide generation.

## Product Thesis

AI tools are already good at generating HTML. The missing layer is a controllable editor and stable document model.

This product should not compete by being a better general AI generator. It should compete by making AI-generated slides editable, reusable, and stable.

## Target Users

Early target users:

1. personal developers building HTML slides
2. AI coding users who generate HTML pages and need to edit them
3. technical writers creating visual explanation pages
4. indie makers creating product decks and landing-page-like slides
5. designers/developers who prefer HTML over PPTX

Avoid early enterprise users. Enterprise workflows introduce collaboration, permissions, compliance, cloud storage, billing, and support overhead.

## Core Jobs To Be Done

### Job 1: Edit slides visually

As a user, I want to edit HTML slides like PPT pages, so I can control layout without editing raw HTML/CSS.

### Job 2: Import AI-generated HTML

As a user, I want to import an HTML file generated by Claude, Codex, GPT, or other tools, so I can continue editing it in a visual editor.

### Job 3: Generate stable output from templates

As a user, I want to generate slides based on fixed templates, so the result is stable and not randomly redesigned every time.

## MVP Feature Set

### P0

- Blank deck creation
- 16:9 slide canvas
- Slide thumbnail navigation
- Text element creation
- Shape element creation
- Select element
- Drag element
- Resize element
- Delete element
- Basic property panel
- localStorage save/load
- Import HTML as sandbox slide
- Export standalone HTML deck

### P1

- Undo/redo
- Duplicate element
- Layer order controls
- Snap-to-grid
- Multi-select
- Alignment tools
- Template selection
- Template slot filling without AI
- Export current slide as PNG

### P2

- AI-generated template slot JSON
- AI HTML-to-SlideDoc conversion
- AI current-slide refinement
- PDF export
- Custom template editor
- Theme presets

## Non-goals

Early versions should not include:

- account system
- database
- cloud sync
- team collaboration
- enterprise workspace
- payment system
- PPTX import/export
- animation timeline
- full Figma-like vector editing

## Product Principle

The product must stay small enough for a personal developer.

Prefer:

```text
small editor + stable schema + clear templates
```

Avoid:

```text
enterprise SaaS + universal design platform + agent workflow platform
```

## Success Criteria for MVP

The MVP is successful if a user can:

1. create a deck
2. create a slide
3. add text and shapes
4. move and resize elements
5. import an HTML page as a slide
6. export the result to standalone HTML
7. reload the project from localStorage

## North Star

Make HTML slides as editable and reusable as PPT slides, while preserving the flexibility of web output.
