# HTML Slide Editor

A lightweight HTML slide editor for the AI generation era.

HTML Slide Editor is not a generic AI PPT generator. It is a structured editing workspace for HTML-based slides:

- edit slides like a lightweight PPT editor
- import HTML files generated by other AI tools and continue editing them
- generate stable slides from templates instead of relying on free-form HTML generation

## Core Positioning

> AI creates the first draft. The editor makes it controllable. Templates make the output stable.

The product focuses on a personal-developer-friendly scope:

- single-user editing
- local-first project state
- HTML slide format
- visual canvas editing
- template-driven generation
- optional AI-assisted creation/refinement

## Why This Project Exists

Current AI tools can generate HTML pages or slide-like layouts, but the result often has three problems:

1. It is hard to edit like a PPT.
2. It is hard to import and reuse across tools.
3. It is unstable when AI freely generates arbitrary HTML.

This project solves those problems by introducing a structured slide document model, visual editor, HTML importer, and slot-based template engine.

## Product Triangle

```text
PPT-like Editor
      +
HTML Importer
      +
Template Engine
      =
Controllable HTML Slides
```

## P0 Scope

The first usable version should support:

- create a blank deck
- create slides with fixed 16:9 canvas
- add text elements
- add rectangle/shape elements
- select, drag, resize, delete elements
- edit text content
- edit basic style properties
- import `.html` file as a sandbox HTML slide
- export deck to standalone HTML
- save/load from localStorage

## Out of Scope for Early Versions

Do not implement these in the early phase:

- login
- database
- team collaboration
- cloud storage
- PPTX import
- PPTX export
- animation system
- template marketplace
- enterprise SaaS features
- complex chart editor

## Recommended Tech Stack

Initial recommendation:

- Vite
- React
- TypeScript
- CSS Modules or Tailwind CSS
- useReducer for editor state
- localStorage for persistence
- optional OpenAI API integration later

## Development Principle

Do not let AI freely generate the final slide structure when stability matters.

Preferred flow:

```text
User input
  -> structured content
  -> template slot values
  -> SlideDoc
  -> renderer
  -> editable canvas
```
